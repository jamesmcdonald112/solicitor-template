# Process Notes — Things to Improve

## 1. Layout System
Claude needs to apply layout systematically — containers, max-widths, 
gutters, and section spacing — consistently across every component.

**Fix:** Add `view _system/LAYOUT-REFERENCE.md` to every build prompt.
The layout reference document defines the exact rules. Once it is in 
context, Claude applies them without being told each time.

---

## 2. Component Direction — Be More Descriptive
Vague prompts produce generic output. Claude needs a specific layout 
concept for every component before it writes any code.

**Fix:** Use the formula from COMPONENT-DIRECTION-GUIDE.md every time:
- Describe the layout concept in plain English
- Specify font-family, font-size, and color tokens per element
- Describe hover behaviour explicitly
- State GSAP requirements if needed
- List hard rules at the bottom

The testimonial component is the benchmark. It worked because the 
concept was clear. Replicate that level of direction for every component.

---

## 3. Pre-Design in Claude Chat Before Building in Terminal
Before running a component session in the terminal, use Claude chat 
(this interface) to work out what the component should look like first.

**Workflow:**
1. Share the relevant config files here in chat
2. Ask Claude chat to suggest a layout concept based on the real content
3. Refine the concept until it feels right
4. Then take that concept into the terminal build prompt

This separates the creative direction (chat) from the execution (terminal),
which is the right division of labour. Terminal Claude executes.
Chat Claude designs.

---

## 4. Build by Component or by Page — Pick One

**By page** (current approach): Build all components for a page in one 
session so the full flow is visible. Good for visual coherence but 
harder to control quality on each individual component.

**By component**: Build one component at a time with full attention. 
Easier to direct and review but slower to see how things work together.

**Recommendation:** Use a hybrid approach:
- Foundation components (Navbar, Footer, Button) → by component
- Page sections → by page, but with layout concepts pre-written 
  for each component before the session starts
- If a component comes out wrong twice → isolate it and build it alone

---

## Summary — The Prompt Sequence Going Forward

Every terminal session should start with:

```
1. view .claude/skills/frontend-design/SKILL.md
2. view _system/LAYOUT-REFERENCE.md
3. view _system/COMPONENT-DIRECTION-GUIDE.md
4. view [relevant config files]
5. view src/styles/tokens.css
6. view src/styles/themes/theme-cream-navy.css
7. view _system/STAGE-3-DESIGN-BRIEF.md
```

Then paste the specific layout concept for each component being built,
written out before the session starts using Claude chat.
